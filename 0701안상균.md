## 시작

- github에 들어가서 Repository를 만듭니다.
- 대시보드에서 new 버튼을 찾아서 저장소 이름을 적습니다
- 그 밑의 Configuration 옵션 - visibility는 공개할지 비공개할지 정하기

## Git 초기화

git init

## 미리 작업해놓은 파일을 추가

git add 0701안상균.md

## 초기 커밋 생성

git commit -m "Initial commit"

## 깃허브에 생성한 레파지토리 주소 연결

git remote add origin git remote add origin https://github.com/emotigom/oz.git

## 기본 브랜치 이름을 main으로 변경

git branch -M main

## 원격 저장소에 코드 업로드

git push -u origin main

## 이후 작업

이제 로컬이 있고, 원격저장소(origin)이 있습니다. 그래서 작업하기 전에 저장소에 최신 내용이 있다면 pull을 해야 하고.
내가 앞서 나가고 있는 상태에서 작업을 시작해서 완료하면 push를 해야합니다.

## 나는 a, b 를 작업 중인데, c의 최신 내용은 가져오고 싶음

1. c의 최신 작업 내용이 내 작업물에 영향을 미친다
   -> 브랜치 다시 만들어야 함

2. c의 최신 작업 내용이 내 작업물에 영향을 안 미친다
   -> pull이 필요 없음

[ GitHub 협업 예시 ]

1)이슈를 생성하고, 담당자 지정 2) 로컬 환경에서 코딩 & 커밋 3) 브랜치 작업 내용을 원격 저장소로 push 4) (main <- branch) Pull Request 생성 5) PR 리뷰
